# תכנון חישוב זכייה לטפסי PIN לפי הגרלה

## מטרה

להוסיף לכל אחד מארבעת טפסי ה-PIN חישוב זכייה כספי עבור ההגרלה העתידית הפתוחה באותו כרטיס. החישוב יתבסס על טבלת הזכיות הרשמית של לוטו רגיל עבור אותה הגרלה, יציג סכום כולל לטופס וסכום לכל קומבינציה, ולא ישנה את מנגנון ה-PIN, תוצאות ההגרלות או חישובי הפגיעות הקיימים.

## החלטות שאושרו

- משתמשים בטבלת **לוטו רגיל** בלבד. דאבל לוטו אינו חלק מהתכונה.
- הסכומים הם סכומי הפרס הרשמיים לפני מס. לא יתבצע חישוב מס.
- החישוב נעשה בנפרד לכל אחד מארבעת הטפסים: טופס ראשון בסיס ומשופר, וטופס שני בסיס ומשופר.
- יוצגו גם סכום כולל לטופס וגם סכום ליד כל קומבינציה.
- ייטענו נתוני פרסים לכל ההגרלות הקיימות ב-`NUMBERS.xlsx`, ולא רק להגרלות חדשות מעכשיו.
- סכום הזכייה הכולל יוצג בפס מודגש מעל ארבעת המדדים הקיימים, כך שמבנה המדדים לא ישתנה.
- מקור הנתונים הוא דף ההגרלה הרשמי בתבנית:
  `https://www.pais.co.il/Lotto/CurrentLotto.aspx?lotteryId=<DRAW_NUMBER>`

## גבולות התכונה

התכונה כוללת:

- איסוף ושמירה של טבלאות זכיות לוטו רגיל.
- מילוי היסטורי לכל ההגרלות שבקובץ התוצאות.
- עדכון אוטומטי של פרסים עבור הגרלות חדשות.
- חישוב זכייה עבור 14 הקומבינציות השמורות בכל PIN.
- הצגת זכייה כוללת ופירוט לכל קומבינציה.
- קישור לטבלת הזכיות הרשמית של ההגרלה.

התכונה אינה כוללת:

- דאבל לוטו, EXTRA או משחק אחר.
- חישוב מס או סכום נטו.
- אימות שהטופס נשלח בפועל למפעל הפיס.
- שינוי בקומבינציות השמורות, בעוגן ה-PIN או בניקוד הקיים.
- משיכת נתוני פרסים ישירות מדפדפן המשתמש.

## ארכיטקטורה

### קובץ פרסים נפרד

יתווסף קובץ סטטי בשם `LOTTO_PRIZES.json`. הוא יישמר ב-GitHub לצד `NUMBERS.xlsx` וייטען על ידי GitHub Pages.

מבנה עקרוני:

```json
{
  "schemaVersion": 1,
  "updatedAt": "2026-07-19T06:00:00Z",
  "draws": {
    "3947": {
      "drawNumber": 3947,
      "drawDate": "18/07/2026",
      "sourceUrl": "https://www.pais.co.il/Lotto/CurrentLotto.aspx?lotteryId=3947",
      "regular": {
        "6+strong": { "winnerCount": 0, "prizeIls": 0 },
        "6": { "winnerCount": 1, "prizeIls": 250000 },
        "5+strong": { "winnerCount": 16, "prizeIls": 8418 },
        "5": { "winnerCount": 119, "prizeIls": 1252 },
        "4+strong": { "winnerCount": 724, "prizeIls": 265 },
        "4": { "winnerCount": 4471, "prizeIls": 85 },
        "3+strong": { "winnerCount": 11352, "prizeIls": 59 },
        "3": { "winnerCount": 65488, "prizeIls": 15 }
      }
    }
  }
}
```

סכומים ומספרי זוכים יישמרו כמספרים, ללא פסיקים או סימן מטבע. המפתח הקנוני של מדרגה יהיה `<MATCHES>` או `<MATCHES>+strong`.

### מפריד אחריות

ייווצר מודול Python נפרד לעדכון פרסים. הוא ישתמש בתוצאות המאומתות של `NUMBERS.xlsx`, אך לא יערוך את קובץ התוצאות. המעדכן הקיים ימשיך להיות אחראי רק לנתוני ההגרלות.

מודול הפרסים יהיה אחראי על:

- הורדת דף ההגרלה הרשמי.
- חילוץ מספר ההגרלה, תאריך ההגרלה ו-`regularLottoList` בלבד.
- נרמול מדרגות, מספרי זוכים וסכומי פרס.
- אימות הנתונים מול מספר ההגרלה המבוקש.
- מיזוג בטוח לתוך `LOTTO_PRIZES.json`.

## מילוי היסטורי ועדכון שוטף

### מילוי ראשוני

מצב `--backfill-all` יעבור על כל מספרי ההגרלות שב-`NUMBERS.xlsx` שאין עבורם נתוני פרסים.

- ההורדות יתבצעו בקצב מבוקר ובאופן סדרתי.
- יהיו ניסיונות חוזרים עם המתנה עולה לכשלי רשת זמניים.
- התקדמות מאומתת תישמר במנות באמצעות החלפה אטומית של JSON תקין.
- הפעלה חוזרת תדלג על הגרלות שכבר נשמרו ותמשיך מהחוסרים.
- בסיום תופק רשימת חוסרים. התהליך ייחשב שלם רק כאשר אין חוסרים.

### עדכון אוטומטי

לאחר עדכון `NUMBERS.xlsx`, המשימה המתוזמנת תריץ את מעדכן הפרסים במצב רגיל:

1. קוראת את מספרי ההגרלות המאומתים מהחוברת.
2. מאתרת מספרי הגרלות חסרים ב-`LOTTO_PRIZES.json`.
3. מורידה ומאמתת רק את ההגרלות החסרות.
4. מחליפה את קובץ ה-JSON באופן אטומי רק לאחר שכל ההגרלות של אותו עדכון אומתו.
5. מפרסמת commit אם `NUMBERS.xlsx`, `LOTTO_PRIZES.json` או שניהם השתנו.

רשימת הקבצים המותרת למשימת האוטומציה תוגבל במפורש לשני הקבצים האלה. שינוי בכל נתיב אחר יעצור את הפרסום.

### שמירת היסטוריה

עדכון רגיל מוסיף הגרלות חסרות ואינו דורס רשומת פרסים קיימת. מצב אימות מפורש יוכל להוריד מחדש רשומות קיימות ולדווח על הבדל, אך שינוי היסטורי לא יפורסם אוטומטית ללא בדיקה.

## טעינת הנתונים באתר

- האתר יטען את `LOTTO_PRIZES.json` עם `cache: no-store` לצד `NUMBERS.xlsx`.
- נתוני הפרסים יישמרו במפה בזיכרון לפי מספר הגרלה.
- כשל בטעינת קובץ הפרסים לא ימנע ניתוח, PIN או השוואת תוצאות.
- לאחר כשל, אזור הזכייה יציג `נתוני זכייה לא זמינים` במקום `₪0` מטעה.
- טעינת קובץ Excel ידני לא תשנה את מקור הפרסים; ההתאמה תמשיך להתבצע לפי מספר הגרלה מול קובץ ה-JSON הציבורי.

## חישוב זכייה

החישוב יבוצע על תוצאות `scorePinnedFormAgainstDraw` הקיימות.

לכל קומבינציה:

1. קוראים את מספר הפגיעות הרגילות ואת מצב המספר החזק.
2. בונים מפתח מדרגה קנוני, לדוגמה `3`, `3+strong` או `6+strong`.
3. מחפשים את המפתח בטבלת הלוטו הרגיל של ההגרלה.
4. אם המדרגה קיימת, הפרס לקומבינציה הוא `prizeIls`.
5. אם המדרגה אינה קיימת, הקומבינציה אינה זוכה.

כל קומבינציה מקבלת לכל היותר פרס אחד לפי המדרגה המדויקת. אין קביעה קשיחה שזכייה מתחילה בשלוש פגיעות; אם טבלה היסטורית מכילה מדרגה אחרת, היא תכובד.

סכום הטופס הוא סכום הפרסים של כל 14 הקומבינציות. מספר הקומבינציות הזוכות הוא מספר הקומבינציות שמצאו מדרגה רשמית עם `prizeIls` גדול מאפס. מדרגה רשמית עם פרס אפס תוצג בשורת הקומבינציה אך לא תיספר כקומבינציה זוכה.

## מצבים מיוחדים

- מדרגה שאינה קיימת בטבלה: `ללא זכייה`.
- מדרגה קיימת עם `prizeIls: 0`: `₪0 · לא חולק`.
- נתוני פרסים חסרים או פגומים: `נתוני זכייה לא זמינים`.
- טבלה ריקה או ללא מדרגות חוקיות אינה נחשבת לטבלה תקינה.
- סכום חסר, שלילי או שאינו מספר פוסל את רשומת ההגרלה.
- מספר זוכים חסר, שלילי או שאינו מספר פוסל את מדרגת הפרס.
- חישוב שאינו זמין אינו מתורגם לאפס ואינו משפיע על מדדי הפגיעות.

## תצוגה

### פס זכייה

בכל `.pinned-future-source` יופיע פס זכייה מעל ארבעת המדדים הקיימים.

מצב זמין:

- כותרת: `זכייה בטופס`.
- סכום בפורמט מטבע ישראלי, לדוגמה `₪89`.
- טקסט משני: `3 קומבינציות זוכות`.

מצב ללא זכייה:

- סכום: `₪0`.
- טקסט משני: `אין קומבינציות זוכות`.

מצב חסר:

- ערך: `נתוני זכייה לא זמינים`.
- ללא ספירת זוכים.

הפס יתעדכן מיד כאשר המשתמש פותח הגרלה אחרת, ורק בכרטיס ה-PIN שבו התרחש האירוע. סגירת כל ההגרלות תחזיר את הפס למצב `פתח הגרלה להצגת נתונים`.

### פירוט קומבינציות

לטבלת 14 הקומבינציות תתווסף עמודת `זכייה`:

- סכום זמין יוצג בירוק.
- מדרגה עם סכום אפס תציג `₪0 · לא חולק`.
- קומבינציה ללא מדרגה תציג `ללא זכייה` בצבע משני.
- נתונים חסרים יציגו `לא זמין`.

מתחת לטבלה יופיע:

`לפי טבלת מפעל הפיס · לוטו רגיל · לפני מס`

הטקסט יקשר ל-`sourceUrl` של אותה הגרלה וייפתח בלשונית חדשה באופן בטוח.

### התאמה למסכים

- במחשב פס הזכייה יציג את הטקסט והסכום באותה שורה.
- בטלפון הפס יעבור לשתי שורות ללא חפיפה.
- ארבעת המדדים הקיימים לא ישנו סדר או גודל בגלל הוספת הזכייה.
- טבלת הקומבינציות תמשיך להשתמש בגלילה אופקית במסכים צרים.
- סכומים ארוכים לא יגדילו את רוחב הכרטיס ולא ייצרו גלילה אופקית של העמוד.

## נגישות ובטיחות

- פס הזכייה יהיה חלק מאזור ה-`aria-live="polite"` של ההגרלה הפתוחה או יקבל אזור חי מקביל אחד בלבד.
- עדכוני DOM יתבצעו באמצעות `textContent` עבור ערכים דינמיים.
- כתובת המקור תיווצר רק מתבנית מפעל הפיס המאושרת וממספר הגרלה חוקי.
- סכומים, מונים ומספרי הגרלה יעברו אימות מספרי לפני הצגה.
- הכיתוב יבהיר שהחישוב הוא לפי טבלת הפרסים ולפני מס, ואינו אישור שהטופס נשלח או ששולמה זכייה.
